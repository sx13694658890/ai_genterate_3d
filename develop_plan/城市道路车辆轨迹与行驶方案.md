# 城市道路车辆轨迹与行驶实现方案

## 1. 背景与目标

当前演示中，车辆通过「路径采样点 + 即时 yaw」驱动，易出现车头与真实前进方向不一致、折线路径上像漂移等问题。本方案按以下原则重新设计：

1. **行驶方向 = 车轮前进方向**（与车身模型局部前向轴严格一致）。
2. **行驶轨迹由城市路面数据生成**（与场景中的道路几何/命名对齐）。
3. **车辆沿轨迹运动**（位置与姿态均由同一条轨迹导出，避免位置与朝向两套逻辑打架）。
4. **转弯平滑**（轨迹与朝向变化均做连续化处理，而不是在折线顶点硬折）。

本文档为**实现方案**（架构与步骤），落地时可对应修改 `ray-interaction` 下的 `Scene.tsx`、`streetDrive.ts`、`Card.tsx` 等模块。

---

## 2. 行驶方向与车轮前进方向一致

### 2.1 概念

- 在 Three.js 中，「车轮滚动轴」与「车身前进方向」在良好绑定的模型里应对齐到同一条**局部前向轴**（常见为 **−Z** 或 **+Z**，取决于导出；需对 `card.glb` 做一次标定）。
- **禁止**再用「世界切线 + 经验常数 `CAR_PATH_YAW_OFFSET`」作为主逻辑；偏移量只能作为**一次性标定**写入配置（例如 `carForwardLocalAxis: '−Z'` 或等价四元数），而不是每帧猜。

### 2.2 推荐做法

1. **标定阶段（离线或首次加载调试）**
   - 在场景中将车放在世界原点、rotation 归零，观察车头指向哪条世界轴。
   - 记录：**局部前向单位向量** `forwardLocal`（在车身根节点或 `Card` 根 `group` 的局部坐标系中）。
   - 将 `forwardLocal` 固化为常量或 Leva 可调项，直到「沿世界 +Z 直线开」时车轮与车身方向正确。

2. **运行时**
   - 由轨迹得到世界空间**切线** `tWorld`（已归一化，投影到 XZ 若只做地面行驶）。
   - 构造目标旋转：使车身的 `forwardLocal` 经世界旋转后与 `tWorld` 对齐（仅绕 Y 可用 `atan2`；若路面有坡需用 `Matrix4.lookAt` 或 `quaternion.setFromUnitVectors` 再抽取 yaw/完整四元数）。
   - **车轮动画**：轮毂角速度 `ω ≈ v / r`（`v` 为沿轨迹弧长速度，`r` 为轮半径估计值），绕**轮轴局部轴**旋转，与 `v` 同号，保证滚动方向与前进一致。

### 2.3 验收

- 直线段：车身、轮胎纹理/条痕与位移方向一致，无侧向滑移观感。
- 无额外「整体绕 Y 神秘 +90°」的魔法数；若有，必须集中在一个标定常量并写注释。

## 3. 城市路面生成运行轨迹

### 3.1 输入数据

- 与现状一致：城市 GLB 内路面片命名前缀 **`road_tile_`**（如 `road_tile_1x1_1.000`）。
- 从已挂载的 `city` 子树中遍历 `Mesh`，`name.startsWith('road_tile_')`，用 **世界包围盒中心** 或 **顶点投影** 得到路面采样点集 `P = {p_i}`。

### 3.2 从点集到「可行驶曲线」

当前「最大连通图 + 贪心 + BFS 闭合」得到的是**折线**，曲率不连续，不适合直接作为「平滑轨迹」源。建议改为两阶段：

| 阶段 | 输出 | 说明 |
|------|------|------|
| A. 拓扑/顺序 | 有序控制点序列 `C = [c_0,…,c_n]` | 仍可用路网邻接（距离阈值连边）、最大连通块、欧拉路径/中国邮递员近似，或手工关键路点 + 自动插中间点 |
| B. 几何平滑 | 参数曲线 `γ(s)` | 在 `C` 上套 **Catmull-Rom 样条** 或 **二次/三次 B 样条**，再 **按弧长重采样** 得到稠密、弧长参数化的折线近似 |

### 3.3 关键参数（建议写入配置）

- 邻接距离阈值（与路块尺寸相关，可据 `road_tile` 最近邻中位数估计）。
- 样条张力 / 分段数；弧长采样步长 `Δs`（越小越顺滑，计算量略增）。

### 3.4 输出接口（实现时建议抽象）

- `buildRoadTrajectory(cityRoot: Object3D): RoadTrajectory`
- `RoadTrajectory` 至少包含：
  - `totalLength: number`
  - `sampleAtDistance(s: number): { position: Vector3; tangent: Vector3; curvature?: number }`  
  其中 `tangent` 为对弧长的导数归一化，保证与「前进方向」一致。

## 4. 让车沿着轨迹行驶

### 4.1 运动学模型（简化）

- 使用**弧长参数** `s(t)`：`s += v * dt`（`v` 为标量速度，可常量或 Leva 调节）。
- 每帧：`pos = γ(s)`，`tangent = γ'(s)`（由 `sampleAtDistance` 提供）。
- 车身 `position` 取 `pos`，必要时 **Y** 用射线投射到路面或 `pos.y + 贴地偏移`，避免穿模。

### 4.2 姿态

- **位置与切线同源**：`rotation` 仅由 `tangent` + 第 2 节标定的 `forwardLocal` 推导，不再单独用另一套 look-ahead 与位置 lerp 混用（避免之前实现中「位置跟一条线、朝向跟另一条线」的错误）。

### 4.3 与现有代码的边界

- `Card` 继续负责模型与轮子网格；**轨迹与 `s` 的积分**放在 `Scene` 或独立 `useStreetDrive.ts` hook 中。
- 城市仍用 `City.tsx` 的 `primitive` 挂载即可；轨迹构建在 `cityRoot` 子树就绪后执行一次（或 road 变更时重建）。

## 5. 转弯平滑

### 5.1 轨迹侧平滑（优先）

- **主手段**：第 3.2 节的样条 + 弧长重采样，使 `γ(s)` 本身 **C¹ 连续**（切线连续），拐点处无突变。
- 可选：**曲率约束**或 **对控制点做 Chaikin 细分** 再拟合，限制最大曲率，避免急弯。

### 5.2 车辆侧平滑（辅助）

在轨迹已较平滑的前提下，再对**朝向**做有限制：

- **最大横摆角速度** `|d(yaw)/dt| ≤ ω_max`，或指数阻尼逼近目标 yaw（时间常数可调）。
- **速度随曲率衰减**（可选）：`v_eff = v * f(κ)`，`κ` 为曲率，弯急自动略减速，更贴近驾驶感。

注意：**不要用强位置滞后去「假装平滑」**，否则会出现车身滞后于轨迹、像漂移；平滑应主要来自**轨迹几何**与**合理的 yaw 限速**。

### 5.3 验收

- 顶点处无明显「车头瞬间甩角」。
- 提高 `Δs` 密度或样条阶数后，转弯应更顺；若仍抖，优先查控制点顺序是否 Zig-Zag 穿越非路面区域。

---

## 6. 实施顺序建议

1. **标定** `card.glb` 前向轴与轮轴（文档化常量）。
2. 实现 **`RoadTrajectory`**：由 `road_tile_` 提取点 → 有序化 → 样条 → `sampleAtDistance`。
3. 用 **`s` 积分 + sample** 驱动车身位置与旋转；轮毂转速接 `v`。
4. 调 **样条/采样** 与 **yaw 限速**，直到直线与弯道均达标。
5. （可选）调试线：`Line` 或 `CatmullRomLine` 可视化 `γ(s)`，便于对比路面。

---

## 7. 风险与回退

- **路网非单连通或顺序差**：贪心路径可能自交或绕远；可回退到「关键路点手工数组 + 自动吸附最近 `road_tile`」的混合模式。
- **性能**：464+ 路块点集上建图 O(n²) 可接受；样条采样建议缓存，每帧只做 O(1) `sampleAtDistance`（二分弧长表）。

---

## 8. 与错误实现的差异小结

| 项目 | 不推荐 | 推荐 |
|------|--------|------|
| 朝向 | 切线 + 经验 `offset` 单独调 | 与模型前向轴标定一致，由轨迹切线算姿态 |
| 轨迹 | 折线顶点直接当路径 | 路面点 → 有序控制点 → 样条 → 弧长参数化 |
| 平滑 | 强依赖位置 lerp / look-ahead 与折线切线混用 | 轨迹几何连续为主，yaw 限速为辅 |
| 车轮 | 与路径速度脱钩仅视觉转 | `ω` 与弧长速度 `v` 绑定 |
