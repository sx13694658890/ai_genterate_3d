Three.js 精灵模型完全指南


引言:什么是精灵模型?
在 Three.js 的3D世界里,Sprite(精灵模型)是一个独特的存在。它不同于我们常用的 Mesh(网格模型),最大的特点就是永远面向相机。无论你怎么旋转场景,精灵模型都会像贴纸一样平行于屏幕,保持正对你的姿态。

这个特性让 Sprite 在某些场景下变得格外有用:制作游戏中的UI元素、模拟雨雪天气、添加场景标注、创建发光特效等等。让我们深入探索这个强大而又轻量的3D对象。

一、精灵模型的核心优势
1.1 性能优势
如果你要模拟10,000个雨滴,如果用真正的3D几何体(比如 SphereGeometry 或复杂的自定义几何体),每个雨滴可能需要40个三角形面片,总共就是40万个面片需要渲染。但是用 Sprite?每个 Sprite 本质上只有两个三角形,10,000个雨滴就是20,000个面片。

性能对比:

真实3D水滴几何体:40个三角形/个 × 10,000 = 400,000个三角形

Sprite模拟水滴:2个三角形/个 × 10,000 = 20,000个三角形

这种性能差异在大规模粒子系统中尤为重要。

1.2 视觉特性
Sprite 最核心的视觉特征是其"面向相机"的特性:

自动调整方向,始终正对视角

适合制作始终可见的UI元素

不受场景旋转影响,保持稳定朝向

二、创建你的第一个精灵模型
2.1 基础创建流程
创建 Sprite 非常简单,只需要一个材质,不需要几何体:

// 导入Three.js库
import * as THREE from 'three';
// 步骤1:创建精灵材质
const spriteMaterial = new THREE.SpriteMaterial({
    color: 0x00ff00,  // 设置精灵颜色(绿色)
});
// 步骤2:创建精灵对象
const sprite = new THREE.Sprite(spriteMaterial);
// 步骤3:添加到场景
scene.add(sprite);
2.2 尺寸控制
Sprite 默认尺寸是 1×1 个单位,在默认设置下可能不够显眼。你可以通过 .scale 属性调整大小:

// 设置精灵的大小
sprite.scale.set(50, 50, 1);  // 宽50, 高50, Z轴保持1
// 动态调整
sprite.scale.x = 100;  // 设置宽度为100
sprite.scale.y = 50;   // 设置高度为50
注意:Sprite 的 Z 轴缩放通常保持为 1,因为它是始终面向相机的平面,Z 轴缩放不会产生视觉效果。

2.3 位置设置
和所有 Three.js 3D 对象一样,可以通过 .position 设置坐标:

// 将精灵放置在三维空间中的某个位置
sprite.position.set(0, 20, 0);  // x=0, y=20, z=0
// 单独设置某个坐标轴
sprite.position.y = 50;  // 只设置Y轴坐标
// 让精灵跟随物体移动
someObject.add(sprite);  // 作为子对象添加到其他3D对象中
三、精灵材质详解
3.1 纯色精灵
最简单的精灵就是使用纯色:

const solidColorSprite = new THREE.SpriteMaterial({
    color: 0xff4444,  // 红色
});
3.2 纹理贴图精灵
更常见的情况是使用图片贴图:

// 加载纹理图片
const textureLoader = new THREE.TextureLoader();
const texture = textureLoader.load('./particle.png');
// 创建带贴图的精灵材质
const texturedSprite = new THREE.SpriteMaterial({
    map: texture,  // 设置颜色贴图
    transparent: true,  // 开启透明(默认已经是true)
    opacity: 0.8,  // 设置透明度
});
3.3 颜色与贴图的混合
如果你的贴图是白色的,你可以通过 .color 属性为精灵着色:

const whiteParticle = new THREE.SpriteMaterial({
    map: whiteTexture,  // 白色粒子贴图
    color: 0x00ffff,  // 混合成青色
});
原理:Three.js 会将贴图颜色和设定的 .color 值相乘,最终显示的是混合后的颜色。

3.4 精灵旋转
虽然 Sprite 始终面向相机,但你可以通过 .rotation 属性旋转精灵平面:

const rotatedSprite = new THREE.SpriteMaterial({
    rotation: Math.PI / 4,  // 旋转45度(弧度制)
});

4.2 动态标注系统
对于需要频繁更新的标注,可以封装一个函数:

function createLabel(text, position, scale = 10) {
    // 这里可以用canvas绘制文字作为贴图
    const canvas = document.createElement('canvas');
    const context = canvas.getContext('2d');
    canvas.width = 256;
    canvas.height = 128;
    // 绘制文字
    context.fillStyle = 'rgba(0, 0, 0, 0.6)';
    context.fillRect(0, 0, 256, 128);
    context.fillStyle = 'white';
    context.font = 'bold 32px Arial';
    context.textAlign = 'center';
    context.fillText(text, 128, 64);
    // 创建纹理
    const texture = new THREE.CanvasTexture(canvas);
    const material = new THREE.SpriteMaterial({
        map: texture,
        transparent: true,
    });
    const sprite = new THREE.Sprite(material);
    sprite.position.copy(position);
    sprite.scale.set(scale, scale * 0.5, 1);
    return sprite;
}
// 使用示例
const pointLabel = createLabel('控制中心', new THREE.Vector3(0, 50, 0), 15);
scene.add(pointLabel);

5.1 雨滴系统完整实现
下面是一个完整的雨滴系统实现:

// 全局变量
const rainGroup = new THREE.Group();
scene.add(rainGroup);
// 雨滴配置参数
const rainConfig = {
    count: 8000,          // 雨滴数量
    rangeX: 1000,         // X轴分布范围
    rangeY: 600,         // Y轴分布范围
    rangeZ: 1000,         // Z轴分布范围
    fallSpeed: 400,      // 下落速度
};
// 加载雨滴贴图
const rainTexture = new THREE.TextureLoader().load('./raindrop.png');
const rainMaterial = new THREE.SpriteMaterial({
    map: rainTexture,
    transparent: true,
    opacity: 0.6,
});
// 批量创建雨滴
for (let i = 0; i < rainConfig.count; i++) {
    const raindrop = new THREE.Sprite(rainMaterial);
    // 随机位置分布
    raindrop.position.x = (Math.random() - 0.5) * rainConfig.rangeX;
    raindrop.position.y = Math.random() * rainConfig.rangeY;
    raindrop.position.z = (Math.random() - 0.5) * rainConfig.rangeZ;
    // 随机大小
    const scale = 2 + Math.random() * 3;
    raindrop.scale.set(scale, scale * 2, 1);  // 雨滴稍长一些
    rainGroup.add(raindrop);
}
// 雨滴动画循环
const clock = new THREE.Clock();
function animateRain() {
    const deltaTime = clock.getDelta();
    rainGroup.children.forEach(raindrop => {
        // 更新Y轴位置(下落)
        raindrop.position.y -= rainConfig.fallSpeed * deltaTime;
        // 如果落到地面以下,重置到顶部
        if (raindrop.position.y < 0) {
            raindrop.position.y = rainConfig.rangeY;
            raindrop.position.x = (Math.random() - 0.5) * rainConfig.rangeX;
            raindrop.position.z = (Math.random() - 0.5) * rainConfig.rangeZ;
        }
    });
}
5.2 雪花系统实现
雪花系统和雨滴类似,但需要调整一些参数:

// 雪花配置
const snowConfig = {
    count: 5000,
    rangeX: 1200,
    rangeY: 800,
    rangeZ: 1200,
    fallSpeed: 100,  // 雪花下落更慢
    windStrength: 50,  // 风力强度
};
// 加载雪花贴图
const snowTexture = new THREE.TextureLoader().load('./snowflake.png');
const snowMaterial = new THREE.SpriteMaterial({
    map: snowTexture,
    transparent: true,
    opacity: 0.8,
});
// 创建雪花群组
const snowGroup = new THREE.Group();
scene.add(snowGroup);
// 批量创建雪花
for (let i = 0; i < snowConfig.count; i++) {
    const flake = new THREE.Sprite(snowMaterial);
    flake.position.x = (Math.random() - 0.5) * snowConfig.rangeX;
    flake.position.y = Math.random() * snowConfig.rangeY;
    flake.position.z = (Math.random() - 0.5) * snowConfig.rangeZ;
    // 雪花大小差异更大
    const scale = 3 + Math.random() * 5;
    flake.scale.set(scale, scale, 1);
    // 存储每个雪花的风力偏移量
    flake.userData = {
        windOffset: Math.random() * Math.PI * 2,  // 初始风力偏移
        rotationSpeed: (Math.random() - 0.5) * 2,  // 旋转速度
    };
    snowGroup.add(flake);
}
// 雪花动画循环
function animateSnow() {
    const deltaTime = clock.getDelta();
    const time = clock.getElapsedTime();
    snowGroup.children.forEach(flake => {
        // 基础下落
        flake.position.y -= snowConfig.fallSpeed * deltaTime;
        // 模拟风吹效果(左右摆动)
        const windX = Math.sin(time + flake.userData.windOffset) * snowConfig.windStrength * deltaTime;
        flake.position.x += windX;
        // 旋转雪花
        flake.material.rotation += flake.userData.rotationSpeed * deltaTime;
        // 重置位置
        if (flake.position.y < 0) {
            flake.position.y = snowConfig.rangeY;
            flake.position.x = (Math.random() - 0.5) * snowConfig.rangeX;
            flake.position.z = (Math.random() - 0.5) * snowConfig.rangeZ;
        }
    });
}

七、进阶技巧与性能优化
7.1 材质复用
创建大量 Sprite 时,一定要复用材质对象:

// ❌ 错误做法:每个雨滴都创建新材质
for (let i = 0; i < 10000; i++) {
    const material = new THREE.SpriteMaterial({ map: texture });
    const sprite = new THREE.Sprite(material);
    // ...
}
// ✅ 正确做法:所有Sprite共享一个材质
const sharedMaterial = new THREE.SpriteMaterial({ map: texture });
for (let i = 0; i < 10000; i++) {
    const sprite = new THREE.Sprite(sharedMaterial);
    // ...
}
7.2 纹理压缩
对于大量使用的纹理贴图,建议使用压缩格式:

使用 .jpg 或 .webp 格式而非 .png(如不需要透明)

控制纹理分辨率,通常256×256或512×512足够

考虑使用纹理数组(TextureArray)实现更复杂的粒子效果

7.3 批量更新优化
在动画循环中批量更新 Sprite 位置时,避免过多的条件判断:

// ❌ 较低效
rainGroup.children.forEach(drop => {
    drop.position.y -= 1;
    if (drop.position.y < 0) {
        drop.position.y = 600;
        drop.position.x = Math.random() * 1000 - 500;
        drop.position.z = Math.random() * 1000 - 500;
    }
});
// ✅ 使用取模运算优化(假设rangeY=600)
const rangeX = 500;
const rangeZ = 500;
const maxY = 600;
rainGroup.children.forEach(drop => {
    drop.position.y -= 1;
    // 使用取模实现循环,避免条件判断
    drop.position.y = ((drop.position.y % maxY) + maxY) % maxY;
});
7.4 视锥体裁剪优化
Three.js 默认会自动裁剪视锥体外的对象,但你可以手动优化:

// 定期清理不可见的Sprite
function cleanupInvisibleSprites() {
    const frustum = new THREE.Frustum();
    const projScreenMatrix = new THREE.Matrix4();
    projScreenMatrix.multiplyMatrices(
        camera.projectionMatrix,
        camera.matrixWorldInverse
    );
    frustum.setFromProjectionMatrix(projScreenMatrix);
    // 检查哪些Sprite在视野外
    rainGroup.children.forEach(sprite => {
        if (!frustum.containsPoint(sprite.position)) {
            sprite.visible = false;  // 隐藏不可见的Sprite
        } else {
            sprite.visible = true;
        }
    });
}

    undefined,
    (error) => {  // 错误回调
        console.error('纹理加载失败:', error);
    }
);
Q2: 如何让 Sprite 在物体后面?

Sprite 默认可能无法正确处理深度关系,可以设置:

const sprite = new THREE.Sprite(material);
sprite.renderOrder = 999;  // 设置渲染顺序
sprite.material.depthTest = false;  // 禁用深度测试
